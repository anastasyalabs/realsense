
# Intel RealSense D415 Calibration and Object Height Detection

## Project overview

This project uses an **Intel RealSense D415** camera to calibrate an empty reference surface and then detect objects placed on that surface. The system captures aligned **depth** and **RGB** frames, builds a reference model of the empty plane, and calculates the **height of objects in millimeters** above that surface.

In this work, I calibrated the RealSense camera by recording multiple depth frames of an **empty surface**, generating a stable reference depth map, and fitting a mathematical plane to that surface. After calibration, the program can compare live depth data against the reference and identify objects that rise above the table.

## What I did

The main task I completed was the **calibration of the Intel RealSense D415 camera**.

The calibration process includes:

* starting the RealSense pipeline with **depth** and **color** streams,
* aligning the depth frame to the color frame,
* warming up the camera so the sensor becomes stable,
* capturing multiple frames of an **empty surface**,
* computing a **median reference depth map** from those frames,
* fitting a **plane model** to the reference surface,
* saving the calibration coefficients and metadata into a JSON file,
* saving the reference depth map into an `.npy` file.

This allows the system to reuse calibration later without repeating the full process every time.

## Goal of the project

The goal of this project is to:

* calibrate a flat working surface,
* measure height differences between the calibrated surface and new scene data,
* segment objects that are placed on the surface,
* visualize detected objects and their approximate height,
* save results for later analysis.

## How the system works

### 1. Camera initialization

The program starts the RealSense camera with:

* depth stream
* color stream
* fixed resolution and FPS

The depth image is aligned to the color image so both data sources match pixel-to-pixel.

### 2. Camera warm-up

Before calibration or measurement, the program reads several frames to stabilize the camera output.

### 3. Calibration on an empty surface

During calibration, the camera must see a **clean empty table or flat surface without objects**.

The program:

* captures several valid depth frames,
* ignores frames with too many invalid pixels,
* stacks the valid frames together,
* computes the **median depth** for each pixel,
* uses this as the reference surface.

### 4. Plane fitting

A plane is fitted to the reference depth map using the equation:

`z = a*x + b*y + c`

Where:

* `x`, `y` are pixel coordinates,
* `z` is the depth value in millimeters,
* `a`, `b`, `c` are the plane coefficients.

The program also calculates:

* **RMSE** – calibration quality / fitting error,
* **valid pixel ratio** – how many depth pixels were usable.

### 5. Height map computation

After calibration, the live depth frame is compared to the reference surface.

The height is computed as:

`height = reference_surface - current_depth`

This means:

* values above zero represent objects above the surface,
* negative values are clipped to zero,
* invalid pixels are ignored.

### 6. Object segmentation

The program detects objects by applying a minimum height threshold in millimeters.

Then it:

* creates a binary mask,
* removes small noise with morphological operations,
* optionally sharpens object boundaries using RGB edges,
* finds contours of valid objects,
* filters out very small contours.

### 7. Object visualization

For each detected object, the program draws:

* object contour,
* bounding rectangle,
* label with object number,
* maximum and average object height.

### 8. Saving results

The program can save:

* RGB image,
* depth matrix (`.npy`),
* height map (`.npy`),
* height map visualization,
* binary mask,
* contour result image,
* calibration JSON,
* reference depth map.

## Files generated by the project

* `calibration.json` – stores plane coefficients and calibration metadata
* `reference_depth.npy` – stores the median reference depth map
* `output/` – contains saved runtime results

## Controls

During runtime, the program supports keyboard commands:

* **C** – calibrate on empty surface
* **S** – save current outputs
* **R** – reload calibration from disk
* **P** – switch between reference depth map and plane mode
* **Q / ESC** – quit the program

## Why this calibration is important

Calibration is the most important part of the system because it defines the reference surface. If the camera is calibrated correctly, the program can reliably:

* detect objects on the table,
* estimate object height in millimeters,
* reduce measurement noise,
* improve contour detection.

Without calibration, the system cannot correctly distinguish the surface from the objects.
